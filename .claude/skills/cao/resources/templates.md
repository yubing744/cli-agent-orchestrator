# CAO任务模板库

## 开发任务模板

### Bug修复模板
```
🔧 任务：修复 [项目名称] 的 [Bug描述]

📋 执行要求：
1. **问题复现**：
   - 在 [项目路径] 中复现问题
   - 确认Bug触发条件和影响范围

2. **根因分析**：
   - 定位问题代码位置
   - 分析产生原因
   - 评估修复复杂度

3. **修复实施**：
   - 设计修复方案
   - 编写修复代码
   - 添加单元测试

4. **验证测试**：
   - 运行现有测试套件
   - 验证修复效果
   - 确保无回归问题

📊 输出格式：
```
=== Bug修复报告 ===

## 问题概述
- Bug描述：[问题]
- 影响范围：[范围]
- 严重程度：[高/中/低]

## 根因分析
- 问题位置：[文件:行号]
- 原因分析：[详细分析]
- 修复策略：[策略描述]

## 修复方案
- 代码变更：
  ```diff
  + 新增代码
  - 删除代码
  ```
- 测试用例：[测试代码]
- 验证结果：[测试结果]

## 风险评估
- 回归风险：[风险评估]
- 兼容性影响：[影响说明]
- 建议部署方式：[部署建议]
```

⏰ 预期时间：[时间估算]
🎯 成功标准：Bug已修复，测试通过，无回归
```

## 使用指南

### 模板选择原则
1. **任务类型匹配**：根据任务性质选择对应模板
2. **复杂度适配**：复杂任务使用详细模板
3. **输出要求**：根据期望输出调整模板格式
4. **时间估算**：参考模板时间进行项目规划

### 模板自定义
- 根据项目特点调整模板内容
- 增加项目特定的检查项
- 调整输出格式符合团队标准
- 添加项目相关的最佳实践

### 最佳实践
- 保持模板的一致性
- 定期更新模板内容
- 收集使用反馈优化模板
- 建立模板版本管理
